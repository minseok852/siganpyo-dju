# 대진대학교 교양선택 강의시간표 이미지 → JSON 변환 프롬프트

당신은 대학교 강의시간표 이미지를 JSON으로 변환하는 작업을 합니다.

## ⚠️ 필수 조건
**이미지에 있는 모든 과목을 빠짐없이 전부 변환해야 합니다. 일부만 하고 멈추지 마세요. 끝까지 완료하세요.**

## 입력
- 교양선택 강의시간표 이미지 (표 형식)
- 영역 번호: [여기에 입력] (예: 1)
- 영역 이름: [여기에 입력] (예: 1영역)
- 분야: [여기에 입력] (예: 인간과소통)

※ 이미지 상단에 "1영역 : 1 / 분야 : 인간과소통" 형태로 표시되어 있음

## 출력 형식

```json
{
  "category": "교양선택",
  "semester": "2026학년도 1학기",
  "updated_at": "2026-01-28",
  "areas": [
    {
      "area_id": 영역번호(숫자),
      "area_name": "영역이름 (예: 1영역)",
      "field": "분야명 (예: 인간과소통)",
      "courses": [
        {
          "course_code": "교과번호 6자리",
          "course_name": "교과목명",
          "credits": 학점(숫자),
          "hours": 강의시간(숫자),
          "lab_hours": 실습시간(숫자, 없으면 0),
          "sections": [
            {
              "section": "분반 2자리 (01, 02 등)",
              "professor": "담당교수",
              "schedule": {
                "raw": "강의시간 원본 (예: 월09:30-11:30)",
                "days": ["월"],
                "times": [
                  {"day": "월", "start": "09:30", "end": "11:30"}
                ]
              },
              "room": {
                "raw": "강의실 전체 (예: 인122-2 - 교양 회화실 1)",
                "building": "건물약칭 (예: 인)",
                "building_full": "건물전체명 (예: 인문학관)",
                "room_number": "호실 (예: 122-2)",
                "room_type": "강의실 유형 (예: 교양 회화실 1)"
              },
              "capacity": 정원(숫자),
              "notes": "비고 내용 (없으면 빈 문자열)"
            }
          ]
        }
      ]
    }
  ]
}
```

## 중요 규칙

### 1. 모든 과목/분반 필수
이미지의 모든 행을 빠짐없이 변환하세요. 중간에 멈추거나 "나머지는 같은 방식으로..." 하지 마세요.

### 2. 같은 과목 다른 분반
같은 course_code면 sections 배열에 추가:
```json
"sections": [
  {"section": "01", "professor": "박선옥", ...},
  {"section": "02", "professor": "박선옥", ...},
  {"section": "08", "professor": "김창남", ...}
]
```

### 3. 교수명이 여러 분반에 걸쳐 있는 경우
이미지에서 교수명이 한 번만 표시되고 여러 분반에 적용되는 경우가 있습니다.
예: 01~07번 분반이 같은 교수 "박선옥"
→ 각 분반에 동일한 교수명을 넣으세요.

### 4. 강의실이 여러 분반에 걸쳐 있는 경우
교수명과 마찬가지로 강의실도 여러 분반에 공통으로 적용될 수 있습니다.
→ 각 분반에 동일한 강의실 정보를 넣으세요.

### 5. schedule 구조
- raw: 원본 텍스트 그대로
- days: 요일만 배열로
- times: 각 시간을 객체로 분리

### 6. room 구조
- raw: 원본 텍스트 전체
- building: 건물 약칭
- building_full: 건물 전체명 (아래 목록 참고)
- room_number: 호실 번호만
- room_type: 강의실 유형

### 7. e-learning/온라인 강의실
온라인 강의의 경우:
```json
"room": {
  "raw": "e-learning(온라인)2",
  "building": null,
  "building_full": null,
  "room_number": null,
  "room_type": "온라인"
}
```

### 8. capacity (정원)
표의 "정원" 열에 있는 숫자를 그대로 입력

### 9. lab_hours
표에서 "실" 열에 있는 숫자. 없으면 0

### 10. 비고(notes)
- "[원격수업]", "간호학과 수강불가" 등
- 줄바꿈 없이 한 줄로

### 11. 홀수/짝수 주의 ⚠️
비고에서 **"홀수", "짝수"**를 절대 **"졸수", "작수"**로 잘못 읽지 마세요.

### 12. 교수명 주의사항 ⚠️
- 한글 이름: 비슷한 글자 주의 (희/의/회, 현/연/원, 수/주/우 등)
- 외국인 이름: 대문자 유지, 띄어쓰기 원본 그대로
- 불확실하면 이미지 원본을 최대한 정확히 작성

### 13. 알파벳/한글 구별 주의 ⚠️
- "AI시대의컴퓨팅사고" → AI는 영문 알파벳
- "사고와표현" → 사는 한글
- "TOEIC", "RC" 등 영어 약자 주의
- 문맥상 영어 약자인지 한글인지 판단할 것

## ⚠️ 최종 확인사항
1. 이미지의 **모든 과목과 모든 분반**을 변환했나요?
2. **영역 정보**(area_id, area_name, field)를 정확히 입력했나요?
3. 같은 과목의 **다른 분반**은 sections 배열에 넣었나요?
4. **교수명/강의실**이 여러 분반에 걸쳐있는 경우 각각 넣었나요?
5. **정원(capacity)** 숫자를 정확히 입력했나요?
6. **온라인 강의**는 room을 null로 처리했나요?
7. **홀수/짝수**를 정확히 읽었나요?
8. **건물 약칭**을 올바른 건물명으로 매칭했나요?

모든 조건을 만족하면 JSON을 출력하세요.
